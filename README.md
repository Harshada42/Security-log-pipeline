# Security-log-pipeline

The purpose of this project is to build a security log data pipeline that takes raw network traffic logs, cleans and organizes them, stores them in a structured format, analyzes them, and then applies machine learning models to identify malicious or abnormal traffic. The project focuses on transforming raw network data into useful security intelligence, since network traffic is often large in volume, complex in structure, and difficult to analyze manually. Therefore, a proper pipeline is necessary to process the data efficiently and use AI or ML techniques to detect suspicious activity.

About Security Intelligence: It is referred to as security intelligence because the pipeline transforms raw network traffic logs into analyzed, structured, and actionable insights that help identify suspicious behavior, detect threats, and support cybersecurity decision-making.

In this project, the security log pipeline ingests and transforms network traffic data for analysis and integrates machine learning models to support anomaly detection and cybersecurity decision-making. SQL was used after loading the cleaned network traffic dataset into MySQL to perform exploratory analysis, including total record verification, label distribution analysis, protocol distribution, and identification of the most frequent destination ports in infiltration traffic.

the IDS dataset was selected, extracted, and one file was chosen initially to keep the work manageable. The data was loaded into Jupyter Notebook, where its shape, columns, missing values, duplicate rows, and label distribution were examined to understand the dataset properly. After that, the data was cleaned by standardizing column names, handling infinite values, and preparing it for both analysis and modeling. Then, a MySQL database was created, the cleaned dataset was stored in a table, and SQL queries were used to analyze the data, including total row count, benign versus infiltration distribution, protocol distribution, and the most frequent destination ports associated with infiltration traffic.

Once the data engineering and SQL analysis stage was completed, the machine learning phase began with the implementation of two models: Logistic Regression as a baseline model and Random Forest as a stronger ensemble model. Both models were evaluated using accuracy, confusion matrix, precision, recall, and F1-score. The results showed that although Logistic Regression achieved slightly higher overall accuracy, Random Forest performed better for infiltration detection because it identified more malicious traffic. Feature importance analysis from Random Forest further revealed that attributes such as destination port, flow timing, flow duration, and packet-rate features played a major role in classification. At this stage, the project already includes data preprocessing, database integration, SQL-based analysis, machine learning implementation, model evaluation, and feature interpretation.
